# QA Portfolio — E-Commerce Application

A hands-on QA portfolio showcasing **UI automation, data-driven testing, manual and API testing, CI implementation**, and comprehensive QA documentation.

---

## About This Project

This repository contains a full-stack e-commerce application built with **HTML, CSS, JavaScript, Express.js, and SQL**.  

The purpose of this project is to **demonstrate practical QA skills** through real-world, hands-on implementations.  

**QA scope includes:**
- Manual testing documentation: test plan, QA test strategy, bug reports, and test execution reports  
- API testing: CRUD operations, boundary value validation, functional/negative/security tests  
- UI automation for critical workflows  
- Continuous Integration for automated test execution (**CI Pipeline Success Screenshot included**)

---

## Key Features

- User flows: **Sign-up, Login, Product Management**  
- Product search and management functionality  
- Automated UI tests for critical workflows  
- API tests for backend endpoints  
- Database seed data and fixture management  
- CI pipeline for automated test execution

---

## Test Automation & Documentation

**UI Automation**
- End-to-end user journey tests using [Playwright]  
- Covers sign-up, login, product management, checkout  

**API Testing**
- Tool used: Postman  
- Functional, negative, and security testing  
- QA artifacts: **API test cases, execution logs, and defect reports**

**Manual Testing**
- Functional, boundary, negative, and security testing  
- Detailed steps, expected results, and test data  
- QA artifacts: **test plan, QA test strategy, bug reports, test execution reports**

**Continuous Integration**
- Automated test execution triggered on each push or PR to the repository  
- Integration with [GitHub Actions]
